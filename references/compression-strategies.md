# 压缩策略详解 · Compression Strategies

---

## 策略总览

| 策略 | 命令 | 适用场景 | 压缩比 |
|------|------|---------|--------|
| summarize | `/hawk compress summarize` | 过程冗长、结论清晰 | 70-90% |
| extract | `/hawk compress extract` | 事实/决策/清单类 | 40-60% |
| delete | `/hawk compress delete` | 临时/调试/过时 | 100% |
| promote | `/hawk compress promote` | learnings/规范类 | 按需 |
| archive | `/hawk compress archive` | 超过30天 | 0%（只是移动） |

---

## 策略1: summarize（摘要）

**原理**：保留结论，删除过程，压缩至核心要点。

**适用**：
- 多轮讨论后的决策记录
- 调试过程的试错记录
- 需求分析的过程记录

**操作步骤**：
1. 读取目标文件
2. 识别"结论"和"过程"
3. 保留结论性语句
4. 删除过程性描述
5. 追加压缩标记

**示例**：

压缩前（80行）：
```markdown
## 2026-03-28 讨论

### 上午讨论
讨论了是否要拆分的skill...
第一轮结论：不拆分，因为...
后来老周说...
又讨论了...
最终决定：不拆分

### 下午
又讨论了电商模块...
决定单独放在 biz/examples/
```

压缩后（15行）：
```markdown
## 2026-03-28 决策记录

### skill结构
- 决定：不拆分，电商作为biz/examples/
- 原因：通用框架+业务示例是正确方向

### 电商模块
- 路径：references/biz/examples/ecommerce.md
- 内容：Enum/DTO/Event/DAO模式完整示例
```

## 策略2: extract（抽取）

**原理**：从大量内容中抽出关键事实/数据/清单，删除冗余描述。

**适用**：
- 包含大量例子的说明
- 长列表式的配置记录
- 包含重复描述的事实记录

**操作步骤**：
1. 识别文件中的核心事实点
2. 提取为清单/表格形式
3. 删除重复的描述性文字

**示例**：

压缩前（60行）：
```markdown
## 团队规范

团队有4个Agent：
- 唐僧：架构师，负责技术方案、规范制定
- 悟空：后端研发，负责Logic/Dao/Model实现
-八戒：前端研发，负责Filament/Blade
- 白龙：测试工程师，负责测试用例

技术规范：
- 四层架构：Controller→Logic→Dao→Model
- 禁止跨层调用
- DTO+Enum必须使用
...
（还有大量重复描述）
```

压缩后（20行）：
```markdown
## 团队规范摘要

| Agent | 角色 | 核心职责 |
|-------|------|---------|
| 唐僧 | 架构师 | 技术方案、规范制定 |
| 悟空 | 后端 | Logic/Dao/Model |
| 八戒 | 前端 | Filament/Blade |
| 白龙 | 测试 | 测试用例 |

核心规范：
- 四层架构禁止跨层
- DTO+Enum强制使用
- 覆盖率≥98%
```

## 策略3: delete（删除）

**原理**：彻底删除指定内容。

**适用**：
- 临时调试记录
- 过时的测试信息
- 已解决的错误记录
- 临时代码片段

**危险操作，需要二次确认**：
```
[Context-Hawk] ⚠️ 危险操作确认
  将删除以下内容：
  - today.md 第 5-15 行（调试日志）
  
  此操作不可逆。
  
  [1] 确认删除
  [2] 只删除今天的内容
  [3] 取消
```

---

## 策略4: promote（晋升）

**原理**：将分散的learnings/规范类内容聚合到对应的主题文件。

**适用**：
- 分散在多处的小技巧
- 重复出现的规范说明
- learnings类内容

**操作**：
1. 扫描所有记忆文件中的 learnings
2. 识别主题归类
3. 聚合到 `memory/learnings.md` 或对应项目文件
4. 在原位置标记 `[已晋升至 learnings.md]`

---

## 策略5: archive（归档）

**原理**：将超期内容移动到 archive/，不影响上下文但不删除。

**适用**：
- 超过30天的内容
- 已完成项目的记录
- 历史版本的规范

**操作**：
1. 检查内容时间戳
2. 移动到 `memory/archive/YYYY-MM/`
3. 在原位置写入跳转标记

## 智能策略选择

让 Agent 分析内容类型后推荐策略：

```markdown
[Context-Hawk] 策略推荐分析

文件：memory/today.md (186行)

内容分析：
- 65% 过程讨论（可summarize）
- 20% 具体事实（保留）
- 10% 调试记录（可delete）
- 5% 重要决策（保留+promote）

推荐策略组合：
1. summarize → 过程讨论（节省 ~100行）
2. delete → 调试记录（节省 ~18行）
3. promote → 重要决策（聚合到week.md）

预计压缩后：68行（减少63%）
```

## 部分压缩

用户可以选择只压缩文件的某个部分：

```
[Context-Hawk] 部分压缩
  today.md 共 186 行

  请输入要压缩的行范围（如 5-50）：
  或输入关键词定位（如 "讨论"）：
```

输入后只压缩指定范围，其余保留。
